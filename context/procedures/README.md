# Procedures

Use this directory as the procedural memory layer of the project.

Procedures should answer:
- how work is typically done in this repository
- which files are usually touched for a recurring change
- what validation should be run before closing the slice

How to use it:
- read a procedure only when the current task matches it
- keep procedures short, concrete, and repo-specific
- link to real files and commands whenever possible
- update a procedure when the project convention changes

Recommended procedure types:
- add or change an API endpoint
- add or change a database schema or migration
- add tests for a new feature
- close a work slice and sync context
- release or deploy a change

Starter procedures in this directory are examples.
Replace them with the real conventions of the target repository after bootstrap.
